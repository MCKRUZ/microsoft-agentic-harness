The Story

This pass designed and built the MediatR pipeline from scratch — deliberately NOT copying the ApplicationTemplate's 6 behaviors verbatim. Instead, each behavior was critically assessed: 2 were removed (redundant), 2 were merged (unified caching), 2 were reworked (Result<T> integration), and 5 new agentic behaviors were designed. The result is a 9-behavior pipeline purpose-built for an agent orchestration template.

Chapter 1: Critical Assessment of Reference Behaviors

Two architecture agents analyzed each of the ApplicationTemplate's 6 behaviors against modern .NET 10 capabilities and agentic requirements. Key verdicts:

Key Design Decision: Result<T> Over Exceptions

Project convention says "Commands return Result<T>, never throw for expected failures." Validation errors, auth denials, and content blocks are expected in an agentic system — agents probe tool boundaries constantly. All behaviors now return Result<T> failures when TResponse supports it, with exception fallback for non-Result handlers.

Chapter 2: Domain Foundation — Result<T>

  • Domain.Common/Result.cs Result + Result<T> + ResultFailureType
  • Domain.Common/Config/AppConfig.cs +AgentConfig section

Result is intentionally a class (not record) because it uses inheritance and value equality is not meaningful for result envelopes. Result<T> is sealed. Implicit conversion from T guards against null. Factory methods: Success, Fail, ValidationFailure, Unauthorized, Forbidden, ContentBlocked.
