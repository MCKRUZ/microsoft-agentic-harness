The Story

This session established the first two concrete layers of the Microsoft Agentic Harness template — starting from an empty repo and building up to a production-quality exception hierarchy and logging infrastructure. The work was driven by two source codebases (ApplicationTemplate for C# patterns, Claude Code for agentic patterns) and went through three review-fix cycles that caught and resolved 25+ issues before any code was committed.

Chapter 1: Project Rules & Memory

Before any code was written, five project rules were established:

  • User approval required — never write code without explicit go-ahead
  • Template mindset — build for future consumers, not just the harness
  • XML docs required — full documentation on all public types
  • Post-change review cadence — /code-review + /review-changes per folder, /simplify per layer
  • File placement rule — always consider project + folder placement via Clean Architecture litmus test
