Key Architectural Decisions

  • In-process over sidecar: Sampling, PII filtering, and rate limiting run as BaseProcessor<Activity> in the OTel SDK pipeline.
  • Order 300 finalization: ObservabilityTelemetryConfigurator runs after all domain sources (Order 100-299).
  • PII-first pipeline: PII filtering runs before rate limiting and sampling so sensitive attributes are scrubbed even from dropped spans.
  • Config POCOs in Domain: All config classes live in Domain.Common/Config/Observability/.
  • Dual deployment: Same behavior available in-process (C# processors) and as standalone OTel Collector.

Beat 3 — Core Processors

Infrastructure Three BaseProcessor<Activity> implementations

Three OTel SDK processors implementing in-process what an external Collector would do.

Files

  • 236 Processors/TailBasedSamplingProcessor.cs
  • 104 Processors/PiiFilteringProcessor.cs
  • 120 Processors/RateLimitingProcessor.cs

TailBasedSamplingProcessor (236 lines)

Buffers spans in ConcurrentDictionary<string, TraceBuffer> keyed by trace ID. Background Timer evaluates aged-out traces. Four-policy cascade: errors, slow requests, agent executions always kept, then probabilistic sampling.

PiiFilteringProcessor (104 lines)

Pre-builds HashSet<string> for O(1) matching. Uses SHA256.HashData static API with Convert.ToHexStringLower.

RateLimitingProcessor (120 lines)

Token bucket with Stopwatch.GetTimestamp(). Dropped spans counted via Interlocked.Increment, summary log every 1000 drops.

Pipeline execution order (within the OTel SDK): Span completed | v [1] PiiFilteringProcessor.OnEnd() -- scrub sensitive attributes | v [2] RateLimitingProcessor.OnEnd() -- drop excess throughput | v [3] TailBasedSamplingProcessor.OnEnd() -- buffer, evaluate, forward/drop | v Exporters (OTLP, Azure Monitor) -- export kept spans
Cross-File Connection
Each processor navigates to its own IOptions<AppConfig> subsection. Configurator snapshots config with Options.Create().
Design Decision
PII filtering runs first: sensitive attributes scrubbed from memory before rate limiting or sampling. Defense-in-depth.
Design Decision
Tail-based sampling uses base.OnEnd(span) preserving the OTel SDK processor chain contract.
